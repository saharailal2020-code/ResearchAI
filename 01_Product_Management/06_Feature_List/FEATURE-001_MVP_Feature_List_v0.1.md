## ResearchAI Platform

**Version:** 0.1

**Status:** Draft

---

# 1. Tujuan Dokumen

Dokumen ini berisi daftar fitur awal ResearchAI untuk tahap Minimum Viable Product (MVP).

MVP adalah versi awal produk yang fokus pada fitur paling penting untuk membuktikan bahwa ResearchAI dapat membantu proses riset dari awal hingga akhir.

Dokumen ini dibuat agar pengembangan ResearchAI tidak langsung melebar ke ratusan fitur sebelum alur utama produk terbukti berjalan.

---

# 2. Prinsip MVP

Pada tahap MVP, ResearchAI harus fokus pada fitur yang:

1. Mendukung proses riset utama.
2. Mengurangi pekerjaan manual.
3. Menghubungkan data antar modul.
4. Memungkinkan penggunaan AI secara praktis.
5. Dapat dikembangkan secara bertahap.

Fitur yang menarik tetapi belum penting untuk membuktikan alur utama produk sebaiknya ditunda.

---

# 3. Modul MVP

## 3.1 Client Management

Tujuan:

Menyimpan dan mengelola data klien sebagai sumber awal proyek riset.

Fitur:

- Menambah data klien
- Mengubah data klien
- Melihat daftar klien
- Melihat detail klien
- Menyimpan kontak utama klien
- Melihat riwayat proyek per klien

Prioritas:

High

---

## 3.2 Proposal Management

Tujuan:

Membantu proses pembuatan dan pelacakan proposal riset.

Fitur:

- Membuat data proposal
- Menentukan nama klien
- Menentukan jenis riset
- Menentukan tujuan riset
- Menentukan metodologi awal
- Menentukan estimasi timeline
- Menentukan estimasi budget
- Melacak status proposal
- Menyimpan file proposal
- Membuat draft proposal dengan bantuan AI

Prioritas:

High

---

## 3.3 Project Management

Tujuan:

Mengelola proyek riset setelah proposal disetujui.

Fitur:

- Membuat proyek dari proposal
- Menentukan project manager
- Menentukan anggota tim
- Menentukan timeline proyek
- Menentukan status proyek
- Melihat daftar proyek
- Melihat detail proyek
- Mencatat progress proyek
- Mencatat catatan penting proyek

Prioritas:

High

---

## 3.4 Survey Management

Tujuan:

Mengelola kebutuhan survei dan fieldwork pada proyek riset.

Fitur:

- Menentukan metode survei
- Menentukan target responden
- Menentukan jumlah sampel
- Menyimpan area survei
- Melacak progress fieldwork
- Mencatat kendala lapangan
- Melihat ringkasan status survei

Prioritas:

Medium

---

## 3.5 Data Processing

Tujuan:

Mengelola data hasil riset agar siap dianalisis.

Fitur:

- Upload file data survei
- Menyimpan dataset per proyek
- Melihat daftar dataset
- Validasi struktur data sederhana
- Menandai data siap dianalisis
- Menyimpan catatan data processing

Prioritas:

High

---

## 3.6 AI Analysis

Tujuan:

Membantu tim riset mendapatkan ringkasan dan insight awal dari data dan dokumen proyek.

Fitur:

- Membuat ringkasan proyek dengan AI
- Membuat draft insight awal
- Membuat draft rekomendasi
- Membuat executive summary awal
- Membantu menyusun narasi temuan utama
- Menyimpan hasil analisis AI sebagai draft

Prioritas:

High

Catatan:

Hasil AI harus selalu dapat direview dan diedit oleh manusia.

---

## 3.7 Report Generator

Tujuan:

Membantu proses penyusunan laporan riset secara lebih cepat dan konsisten.

Fitur:

- Membuat draft struktur laporan
- Membuat narasi laporan berdasarkan hasil analisis
- Membuat executive summary
- Membuat rekomendasi awal
- Menyimpan draft laporan
- Melakukan revisi manual pada draft laporan

Prioritas:

High

---

## 3.8 User and Access Management

Tujuan:

Mengatur pengguna dan hak akses dasar di dalam ResearchAI.

Fitur:

- Login pengguna
- Logout pengguna
- Menyimpan data pengguna
- Menentukan role pengguna
- Membatasi akses berdasarkan role

Prioritas:

High

---

## 3.9 Dashboard

Tujuan:

Memberikan ringkasan kondisi proyek dan aktivitas utama ResearchAI.

Fitur:

- Melihat jumlah proyek aktif
- Melihat jumlah proposal berjalan
- Melihat status proyek
- Melihat progress fieldwork
- Melihat proyek yang membutuhkan perhatian

Prioritas:

Medium

---

# 4. Fitur Yang Ditunda Setelah MVP

- Advanced CRM
- Mobile Enumerator App

- Advanced Data Visualization
- Notification System
- Client Portal
- Vendor Management
- Template Marketplace
- Multi-company Support
- Advanced Audit Log
- API Public Access

Fitur tersebut dapat dikembangkan setelah alur utama ResearchAI terbukti berjalan.
